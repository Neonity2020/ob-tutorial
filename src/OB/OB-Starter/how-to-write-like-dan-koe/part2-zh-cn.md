# 第二部分：如何建立你的第一个AI创意团队

#### *从零散的提示词到思考系统*

大多数人在使用AI时失败的原因只有一个：

他们把智能当成自动售货机。

* 输入提示。
* 获得输出。
* 重复。

那不是团队。

那是噪音。

真正的创意团队拥有：

* 角色
* 边界
* 反馈循环
* 共同的方向

你的AI系统需要同样的结构。

本文将一步步指导你建立第一个功能性的AI创意团队。

## 第一步：从单一方向开始，而不是多个目标

在考虑代理、工具或提示之前，你需要一样东西：

> 清晰的创意方向

而不是：

* "我想在Twitter上成长"
* "我想写得更好"
* "我想通过内容赚钱"

而是更接近：

* "我帮助独立创作者进行系统化思考。"
* "我写关于杠杆、身份和长期思考的内容。"
* "我的内容是[[复利]]的[1]，不是追逐[[潮流]]。[2]"

AI无法创造清晰度。

它只能放大已经存在的东西。

如果你的方向是[[模糊的]][3]，你的智能体也会是模糊的。

用一段话写下来。这将成为你的**北极星提示词**。[4]

## 第二步：在写提示之前先设计角色

大多数人直接跳到写提示。

这是本末倒置。

在真实组织中，你不会雇佣一个"提示词"。你雇佣的是负责的人。

从3-5个核心角色开始。超过这个数量会创造复杂性而失去杠杆效应。[5]

一个简单的创始团队：

### 创意总监（你）
愿景、判断、最终决策。

### 编辑智能体
清晰度、结构、语调。

### 想法智能体
原始概念、角度、重新框架。

### 系统智能体
长期一致性和知识结构。

### 分发智能体
标题、钩子和格式适配。

每个智能体的存在都是为了以有用的方式与你产生分歧。

## 第三步：给每个代理一个永久身份

没有身份的代理只是自动补全。

每个角色都需要：

* 一个世界观
* 一份偏见
* 一份工作描述

### 示例：编辑智能体

```
* 你是一个无情的编辑。
* 你优先考虑清晰度而非聪明。
* 你移除任何不能推进想法的内容。
* 你为有思想、聪明的读者进行优化——不是为了病毒式传播。
```

### 示例：想法智能体

```
* 你产生非常规的角度。
* 你挑战默认假设。
* 你更喜欢第一性原理思考而非趋势。
```

保存这些身份。

重复使用它们。

这是代理随时间保持一致性的方式。

## 第四步：将思考与写作分离

这是大多数创作者遇到[[瓶颈]]的地方。[6]

他们要求AI：

> "写一篇关于X的文章。"

相反，将创作分解为阶段。

### 阶段1：思考

* 想法智能体生成概念
* 系统代理检查与长期主题的一致性

### 阶段2：结构化

* 编辑代理建立逻辑大纲
* 你批准或调整方向

### 阶段3：表达

AI基于批准的结构写作

编辑润色语言

你不是外包创造力。

你是在模块化认知。

## 第五步：创建反馈循环

团队因为反馈而改进。

你的AI团队也需要它。

发布后：

* 什么产生了共鸣？
* 什么感觉不对？
* 你不完全同意什么？

将这些反馈回你的代理：

* "写作时，避免励志陈词滥调。"
* "假设读者已经很聪明。"
* "优先考虑深度而非广度。"

随着时间的推移，你的代理开始听起来不再像AI，而更像你思维的延伸。

## 第六步：构建知识骨干

没有记忆，你的系统每次都会重置。

创建一个单一文档（或文件夹）包含：

* 你的核心信念
* 反复出现的主题
* 你回归的最喜欢的比喻/模型
* 这成为你的代理参考的骨干。

内容不再成为孤立的帖子，而是开始成为作品体系。

## 第七步：只有在有摩擦时才扩展

不要因为感觉强大而添加代理。

当有东西损坏时才添加它们。

### 示例：

* 在变现方面挣扎 → 添加业务智能体
* 故事叙述薄弱 → 添加叙事智能体
* 语调不一致 → 加强你的编辑身份

复杂性应该遵循必要性。

而不是好奇心。

## 你真正在构建什么

你不是在构建提示词。

你在构建：

**外化的思考系统**

> AI是媒介。你是源头。

随着时间的推移：

* 你的想法变得敏锐
* 你的产出复利增长
* 你的努力减少

这就是一个人的杠杆看起来像的样子。

* 不是忙碌。
* 不是自动化。
* 而是设计智能。
